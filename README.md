# Case Lab 1С: Конфигурация для процессов закупки и продажи

## О проекте

Разработана конфигурация на платформе **1С:Предприятие** для автоматизации бизнес-процессов "Закупки" и "Продажи" для кафе. Проект выполнен в рамках предстажировки **Case Lab 1С** ГК «Росатом» (АО «Гринатом»)

## Технологии
- **Платформа:** 1С:Предприятие 8.3
  
## Основные бизнес-процессы

### 1. Закупки
- Ввод информации об организации, поставщиках, номенклатуре (ингредиенты, расходные материалы)
- Планирование поставок с контролем кратности закупки (например, упаковками по 100 шт.)
- Оформление поступления товаров
- Печатная форма документа плановой поставки
- АРМ закупщика с визуальным выделением просроченных поставок
- Отчёт об остатках запасов в количественном выражении

### 2. Продажи
- Ввод информации о клиентах и продаваемой номенклатуре (включая технологические карты)
- Оформление заказов клиентов с учётом вида обслуживания (доставка / зал)
  - При доставке - обязательный ввод адреса
  - Для зала - адрес не требуется
- Отражение продажи блюд
- Печатная форма «Чек покупателю» с рекламным дизайном
- Отчёт по работе курьерских служб за произвольный период (количество и сумма доставок)

## Дополнительные возможности

1. **Настройка списка закупок в АРМ закупщика**
   - Фильтрация: не отображать реализованные, только просроченные, только запланированные, за определённый период

2. **Email-уведомления**
   - Автоматическое уведомление поставщика о нарушении сроков поставки
   - Уведомление при создании новой потребности в ингредиентах или расходных материалах

3. **Чек лояльности (система скидок)**
   - Расчёт скидок для клиентов
   - Вывод информации о скидках в чек покупателя (расширенная форма чека)


## Как развернуть конфигурацию
1. Зайдите в информационную базу или создайте новую пустую информационную базу
2. Запустите выбранную базу в режиме «Конфигуратор»
3. В верхнем меню выберите «Администрирование», далее «Загрузить информационную базу»
4. Выберите .dt-файл
